CSS3 tarjoaa ominaisuuksia, joita ei ole aikaisemmissa versioissa.

Sivussa tulee olla otsikoituna ja esiteltynä (kerrottuna kuinka se näkyy sivulla) valittu ominaisuus, ominaisuuteen sopiva esimerkki ja luettelo selaimista, joissa ominaisuus on käytettävissä.

  2. 3d-liike(rotate) - huom. jos haluat kokeilla liikettä, toimii kohdan animate kanssa, toinen vaihtoehto on JavaScript
  3. animointi
  4. ominaisuuden muuttuminen (transition)
  5. canvas, sen käyttömahdollisuudet

GitHubin käyttö yhdessä

Case 1: Molemmat muokkaavat Main/Master haaraa

Jompikumpi perustaa uuden repositoryn (esim. co-op-demo.github.io)
ja jakaa repositoryn toiselle käyttäjälle: Settings > Collaborators > Add people

Kloonatkaa repo tietokoneelle:
git clone {repo URL} Tee muokkauksia projektiin...
Avaa komentorivi/CMD projektikansiossa: git add -A git commit -am"kuvaus muokkauksista" git push jos saat viestin "error: failed to push some refs to…" se tarkoittaa,
että toinen käyttäjä on tehnyt muutoksia repoon tällä välin.

Anna komento: git pull Nyt paikallinen kansiosi on ajan tasalla.
Katselmoi mitä koodissa oli ristiriidassa ja poista turhat huomautukset.

Kokeile uudestaan: git add -A git commit -am"kuvaus muokkauksista" git push Jos Auto-merging herjaa jotain, niin sulje sen avaama tiedosto.

Case 2: Molemmilla on oma kehityshaara (branch)

Voitte myös kumpikin tehdä repositoryyn oman kehityshaaran (branch),
jolloin toisen henkilön tekemät muutokset eivät vaikuta toisen tekemisiin. git checkout -b {branch-name} git add -A git commit -am"kuvaus muokkauksista" git push -u origin {branch-name} Nyt näet kehityshaarat GitHub repon sivulla 'branches' -nappia painamalla.

Kun jatkat koodin muokkaamista ja haluat ladata taas muutokset branchiin,
et enää tarvitse uuden branchin luontikomentoa (git checkout -b {branch-name})

Lähetä tekemäsi muutokset Gittiin:
git add -A git commit -am"kuvaus muokkauksista" git push -u origin {branch-name} Onnistuitteko?
-Ei mekään TAMKilla ainakaan ekalla kertaa!

Jos kotikoneella palaat muokkaamaan branchia niin
lataa master/main -haaran sijaan branch komennolla:
git clone --branch {branch-name} {remote-repo-url} {target-dir} esim.: git clone --branch mun-oksa https://github.com/OmaNimi/co-op-demo.github.io kansion-nimi
